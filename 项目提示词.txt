== 已知踩坑（别再踩） ==
1. 后端 CORS_ORIGIN 环境变量一旦设置，就完全替代源代码里的正则兜底。
   当前值要包含: https://jujuju-28b.pages.dev,https://www.ddddkiii.com,https://ddddkiii.com
   加新域名要追加进去，别覆盖。
2. 前端处理登录响应字段名是 accessToken / refreshToken，不是 token。
   涉及文件: frontend/src/store/user.ts, src/types/index.ts, src/api/auth.ts。
   改 auth 相关类型时这三个文件要在同一个 commit 里推（用 Git Trees API），
   单文件 PUT 会导致 TS 编译失败、CF 构建挂掉。
3. 前端 Pinia 用 sessionStorage 持久化，key 是 matrixflow-user。
   改 auth 流程后让用户「关闭标签页再重开」，光刷新清不掉。
4. 后端这些接口前端调了但还没实现（返回 404，需要时要我加）：
   GET /api/v1/auth/me, POST /api/v1/auth/profile
5. frontend/src/views/dashboard/DashboardView.vue 的数字（128 账号、2.8M 粉丝
   等）全是写死的 mock 数据，onMounted 是空的，还没接真实 API。
6. 密码规则: 至少 8 位 + 含大写字母 + 含数字（如 Test12345）。
7. 注册成功后不会自动登录，会切换到登录 tab。
