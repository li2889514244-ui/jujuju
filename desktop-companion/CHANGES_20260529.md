# 披星云桌面伴侣 v3.1 → v3.2 修改记录

**日期**：2026-05-29  
**文件**：`companion_app.py`、`chrome_cdp.py`、`披星云伴侣.spec`

---

## 修改 1：砍掉启动时自动采集

**位置**：`companion_app.py` 旧 1776-1777 行

**问题**：`_data_collector_loop` 在模块加载时（`import` 阶段）就启动后台线程，30 秒后自动对所有已绑定账号开窗采集。用户启动后看到的不是登录界面，而是满屏的数据采集窗口。

**修改**：
- 删除模块级 `threading.Thread(target=_data_collector_loop, daemon=True).start()`
- 新增全局标志 `_collector_loop_started = False`
- 在 `/api/data-collection/trigger` 中首次触发时才启动后台定时循环

**影响**：启动后不再自动采集。用户需在 UI 点击"触发采集"按钮，之后后台按白天30分钟/夜间2小时自动轮询。

---

## 修改 2：数据采集改用独立 headless CDP Chrome

**位置**：`companion_app.py` 函数 `_scrape_all`

**问题**：旧代码 `_scrape_all` 连接到用户可见的 CDP Chrome（端口 9222），每个账号 `new_context()` 都开一个可见 Chrome 窗口，导致窗口泛滥。

**修改**：
- `_scrape_all` 不再连接可见 CDP Chrome (9222)
- 改为启动**独立的 headless CDP Chrome**（端口 9223，`--headless=new`）
- 用真实 Chrome 二进制（`_BROWSER_PATH`），保真浏览器指纹
- 采集完成自动 `terminate()` 回收进程
- 增加 `try/finally` 确保进程一定被清理

**架构**：
```
可见 Chrome (9222)  ← UI + 扫码绑定
无头 Chrome (9223)  ← 数据采集专用（真指纹，零窗口）
```

---

## 修改 3：context.clear_cookies() 防崩溃

**位置**：`companion_app.py` `_make_login_worker` 内层函数 `_run`

**问题**：日志报 `TargetClosedError: BrowserContext.clear_cookies`。CDP Chrome 的 context 可能被用户关闭后，扫码绑定时对已失效 context 调 `clear_cookies()` 崩溃。

**修改**：
- `connect_over_cdp` 和 `context.clear_cookies` 加 try/except
- `clear_cookies` 失败时自动 `close()` 旧 context，重建新 context
- 所有 `page.close()` 加 try/except 兜底

---

## 修改 4：_scrape_account_pages 容错

**位置**：`companion_app.py` 函数 `_scrape_account_pages`

**修改**：
- `context.new_page()` 加 try/except，context 关闭时返回空结果
- `page.close()` 加 try/except，忽略已关闭的 page

## 修改 5：/api/config 敏感信息过滤

**位置**：`companion_app.py` `/api/config` 路由

**问题**：返回 `refreshToken`，可被前端 JS 读取。

**修改**：过滤列表增加 `refreshToken`、`accessToken`，不返回给前端。

| 启动 | 1 个 Chrome 窗口显示登录界面 |
