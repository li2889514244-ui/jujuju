- 所有业务模块遵循标准的 NestJS 三段式结构：Controller 处理请求路由，Service 封装核心逻辑，Module 负责依赖注入与组件聚合。
- API 响应通过 `TransformInterceptor` 统一包装为 `{ code, message, data, timestamp }` 格式，成功状态码固定为 0。
- 敏感配置（如 JWT 密钥、数据库连接）通过 `@nestjs/config` 的 `registerAs` 模式加载，并在配置文件中执行严格的格式与长度校验。
- 跨模块共享的服务（如 PrismaService, RedisService, PlatformsService）在各自的 Module 中通过 `exports` 导出，并在根模块或消费模块中导入。
